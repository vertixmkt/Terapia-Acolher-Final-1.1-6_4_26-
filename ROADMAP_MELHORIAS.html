Prioridades
As 6 Prioridades do Sistema

Definidas na call de 13/03/2026. Todas implementadas no código do backend novo.

1
Fluxo de entrada do terapeuta via Kiwify
Compra → cadastro → fila automática com saldo correto
Implementado
  • Webhook da Kiwify registra compra e detecta quantidade de leads pelo nome da oferta
  • Terapeuta preenche formulário público de cadastro
  • Ao cadastrar, sistema vincula automaticamente webhooks Kiwify pendentes e credita saldo correto
  • Rodrigo aprova → terapeuta entra na fila ativa com saldo real (não mais fixo em 10)
  • Aguardando: configurar URL do webhook no painel Kiwify
2
Matching automático com filtros eliminatórios
Exigências do paciente vêm primeiro, independente da posição na fila
Implementado
  • Filtros eliminatórios: gênero do terapeuta (se paciente pede M ou F), turno, saldo > 0, status ativo
  • Dentro dos elegíveis: ordenação por quem está há mais tempo sem lead (fila justa)
  • Score de compatibilidade: especialidades + abordagem vs. queixa do paciente
  • Modos: Automático / Semi-auto / Manual / Pausado
3
Reposição de leads (lead fantasma)
Terapeuta solicita até 3 reposições por ciclo
Implementado
  • Terapeuta abre solicitação informando protocolo de contato (0h, 24h, 72h, 15 dias)
  • Sistema verifica limite máximo de 3 reposições por ciclo
  • Admin aprova → 1 lead é creditado automaticamente
4
Lista de webhooks recebidos do ManyChat
Todos os dados que chegam do ManyChat ficam registrados
Implementado
  • Nome, telefone, gênero, turno, motivo, subscriber ID de cada paciente recebido
  • Status de processamento (processado / erro) com payload completo
  • Endpoint: GET /api/webhooks/manychat/received
  • Pendente: página no frontend para visualizar esta lista
5
Lista de webhooks enviados ao ManyChat
Notificações para paciente e terapeuta todas rastreadas
Implementado
  • Notificação ao paciente (dados do terapeuta atribuído) e ao terapeuta (dados do novo paciente)
  • Status de cada envio (success / error / skipped) com payload e resposta do ManyChat
  • Endpoint: GET /api/webhooks/manychat/sent
  • Pendente: botão de re-envio em caso de erro + página no frontend
6
Lista de webhooks recebidos da Kiwify
Todas as compras rastreadas com status de vinculação
Implementado
  • Nome, e-mail, produto, oferta, leads e valor de cada compra
  • Status: pendente (terapeuta ainda não cadastrou) / processado / erro
  • Terapeuta vinculado quando identificado pelo e-mail ou telefone
  • Endpoint: GET /api/webhooks/kiwify
  • Pendente: página no frontend + botão para processar pendentes manualmente

Diagnóstico
Estado atual do sistema

Com base no relatório fornecido, identificamos os seguintes problemas.

Problemas Críticos
  • 742 pacientes sem terapeuta atribuído — fila parada
  • 4.798 atribuições com terapeuta inativo — dado sujo em produção
  • Links públicos de cadastro não funcionam — bloqueando entrada de novos terapeutas
Problemas Funcionais
  • Dashboard não apresenta informações úteis
  • Subscriber ID do terapeuta no ManyChat não é capturado automaticamente
  • Não existe área de acesso para o terapeuta acompanhar seu perfil
  • Validação de compra Kiwify antes de liberar cadastro não implementada
Código Legado (será removido)
  • Integração Z-API (abandonada, derrubava chips)
  • Telas: Monitoramento Fila, Análise Mensalidades
  • Telas: Métricas Terapeuta, Comparação Envios
  • Telas: Auditoria, Feedback, Debug Logs, Histórico Z-API

3
Portal do Terapeuta
Semana 5
Nova Funcionalidade UI Concluída · Backend pendente
O terapeuta tem autonomia para gerenciar seus próprios dados sem precisar do admin.
  • Login via link único enviado pelo ManyChat (token temporário) (aguarda backend)
  • Tela "Meu Perfil" — editar dados, turnos, abordagem, público atendido
  • Tela "Minhas Atribuições" — lista de pacientes recebidos e histórico
  • Tela "Meu Saldo" — histórico de compras, saldo atual, link para comprar mais
  • Restrição total — terapeuta só acessa seus próprios dados (aguarda autenticação real)
👤 Entregável: Portal do terapeuta funcional, seguro e com autenticação separada da área admin
4
Algoritmo de Matching
Semana 6
Inteligência UI Concluída · Backend pendente
Matching mais preciso, com controle fácil pelo admin — sem precisar de desenvolvedor para ajustar.
  • Auditoria do algoritmo real — arquivo matching-23-02-v3.ts lido e mapeado
  • Correção dos casos onde pacientes ficam presos sem atribuição (aguarda acesso ao banco)
  • Painel de controle — ajustar pesos dos critérios (gênero, turno, abordagem, IA)
  • Log visual de cada decisão do algoritmo — por que X foi atribuído a Y
  • Matching semi-automático: admin seleciona paciente → IA sugere terapeuta com score + motivo
  • Matching manual com seleção de terapeuta diretamente na ficha do paciente
🧠 Entregável: Algoritmo revisado, zero pacientes presos, admin controla critérios sem precisar de dev
5
Polimento Final
Semana 7
Entrega Pendente
Sistema pronto para operar de forma autônoma, sem suporte técnico constante.
  • Revisão de UX das telas de Pacientes, Terapeutas e Atribuições
  • Exportação de atribuições para Excel com filtros por período e status
  • Logs ManyChat com re-envio fácil em caso de erro
  • Testes de carga com 10k+ pacientes e múltiplos usuários simultâneos
  • Documentação de operação para o time administrativo
🎯 Entregável: Sistema estável, documentado e time treinado para operar sem suporte técnico
